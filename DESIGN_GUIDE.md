# QuickBite – Creative Design Guide

## Using Google Stitch MCP + 21st.dev MCP

This document defines the UI/UX design strategy for building a modern and professional interface for QuickBite.

Goal:
Create a clean, premium-looking restaurant ordering system that feels like real restaurant management software instead of a basic college project.

---

# Design Philosophy

## Core Principles

- Minimal but premium
- Fast and smooth user flow
- Strong visual hierarchy
- Restaurant-style modern interface
- Professional internship-ready design
- Easy navigation
- Clean spacing and alignment

Avoid:

- Overloaded screens
- Too many colors
- Small unreadable buttons
- Old-style school project layouts

---

# Recommended Design Theme

## Style Direction

Use:

- Soft rounded cards
- Clean typography
- Modern dashboard style
- Large CTA buttons
- Minimal shadows
- Consistent spacing
- Premium food ordering feel

Inspiration:
Restaurant POS software + modern delivery app UI

---

# Color Strategy

## Primary Colors

Use a premium restaurant palette:

Primary Accent:
Warm orange / rich red

Secondary:
Soft cream / white

Support:
Dark charcoal / muted black

Highlight:
Fresh green for success states

Avoid:
Bright neon colors

---

# Typography Guide

## Heading Style

Use bold large titles

Examples:

- Welcome to QuickBite
- Your Cart
- Billing Summary

## Body Text

Use clean readable medium-sized text

## Buttons

Use strong clear labels

Examples:

- Start Order
- Add to Cart
- Generate Bill

# Screen-by-Screen Design Guide

# 1. Home Page

## Goal

Strong first impression

## Required Elements

- Logo area
- Project title
- Restaurant branding
- Welcome message
- Start Order button

## Design Suggestion

Large centered hero section
with premium restaurant branding

---

# 2. Customer Details Page

## Goal

Simple and clean form

## Required Fields

- Name
- Mobile Number
- Address

## Design Suggestion

Card-based form layout
with clear spacing and validation

---

# 3. Food Menu Page

## Goal

Beautiful product browsing

## Required Layout

Use card layout for food items:

Each card should include:

- Food image placeholder
- Item name
- Price
- Quantity selector
- Add to Cart button

## Design Suggestion

Grid-based responsive layout
with attractive food cards

---

# 4. Cart Page

## Goal

Fast order review

## Required Elements

- Selected items
- Quantity controls
- Remove button
- Subtotal
- GST
- Final Total

## Design Suggestion

Split layout:
Left = cart items
Right = billing summary

---

# 5. Billing Page

## Goal

Professional final bill

## Required Elements

- Customer info
- Ordered items
- Tax summary
- Final total
- Generate Bill button
- Print Receipt button

## Design Suggestion

Invoice-style layout
clean professional receipt design

# UX Optimization Rules

## Important

Every page must answer:
What should the user do next?

Use clear CTA buttons.

Example:
Menu Page → Add to Cart
Cart Page → Proceed to Billing
Billing Page → Generate Bill

Never create confusion.
